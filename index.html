// Errores comunes de principiantes

Estos son los errores más frecuentes al aprender Godot 4 y GDScript. Reconocerlos te ahorrará horas de debugging.

⚠️ 1. No multiplicar por delta

El error más común. Si mueves un objeto sin usar delta, el movimiento será diferente en cada pantalla (60 fps vs 144 fps).

GDScriptmal.gd
# ❌ MAL: el movimiento depende de los fps
func _process(delta):  # También debería ser _physics_process
    position.x += 5

# ✅ BIEN: frame-rate independiente
func _physics_process(delta: float) -> void:
    position.x += 200 * delta

⚠️ 2. Usar get_node() con rutas quemadas

Llamar $Nodo cada frame es ineficiente. Además, si renombras el nodo, el código se rompe.

GDScriptmal.gd
# ❌ MAL: get_node cada frame + ruta quemada
func _process(delta):
    var sprite = $Sprite2D
    sprite.modulate = Color(1, 0, 0)

# ✅ BIEN: @onready cachea la referencia
@onready var sprite: Sprite2D = $Sprite2D

⚠️ 3. No desconectar señales

Si conectas una señal en código pero no la desconectas al salir de la escena, obtienes errores de "null instance" o comportamientos duplicados.

GDScriptmal.gd
# ❌ MAL: señal nunca se desconecta
func _ready() -> void:
    enemy.hit.connect(_on_enemy_hit)

# ✅ BIEN: desconectar en _exit_tree
func _ready() -> void:
    enemy.hit.connect(_on_enemy_hit)
func _exit_tree() -> void:
    enemy.hit.disconnect(_on_enemy_hit)

⚠️ 4. Un solo script gigante

Poner toda la lógica del juego en un solo archivo hace que el código sea imposible de mantener. Godot está diseñado para composición de nodos.

Regla práctica: Si un script supera las 200-300 líneas, dividí su comportamiento en nodos hijos especializados (HealthComponent, MovementController, etc.).

⚠️ 5. No usar tipos (type hints)

GDScript 2.0 soporta tipado estático opcional. No usarlo significa perder autocompletado, detección de errores en tiempo de compilación y rendimiento.

GDScriptmal.gd
# ❌ MAL: sin tipos
func take_damage(amount, source):
    return hp - amount

# ✅ BIEN: con tipos
func take_damage(amount: int, source: Node) -> int:
    return hp - amount

Bienvenida // El entorno Godot
🔰 En palabras simples

Un motor de videojuegos es un programa que ya trae resueltas las partes difíciles de hacer un juego: dibujar imágenes en pantalla, detectar choques, reproducir sonidos y aplicar física. Godot es uno de esos motores, gratis y libre. En vez de programar todo desde cero, armás tu juego combinando piezas (los nodos) y escribiendo pequeñas instrucciones en GDScript, un lenguaje pensado para leerse fácil. No necesitás experiencia previa en videojuegos para empezar.

Analogía: pensá en Godot como una cocina ya equipada: el horno, los cuchillos y las ollas están. Vos ponés la receta (tu código) y los ingredientes (imágenes y sonidos).

Godot 4 es un motor de videojuegos open source con un lenguaje propio llamado GDScript, diseñado para ser legible, rápido y muy integrado con el editor. Su sintaxis es similar a Python pero con tipos opcionales y acceso directo al engine.

Instalación // Setup del proyecto
🔰 En palabras simples

Instalar Godot es más simple que la mayoría de programas: no hay instalador, descargás un solo archivo y lo abrís. Lo primero que vas a hacer es crear un proyecto, que no es más que la carpeta donde vivirá tu juego. Todo lo que crees se guarda ahí adentro.

⚠️ Elegí una ruta sin espacios ni tildes (por ejemplo C:/godot/mi_juego): algunos sistemas se confunden con caracteres raros en la ruta.

Analogía: crear un proyecto es como abrir una carpeta nueva en blanco para un trabajo: todavía no hay nada, pero es el lugar donde irás poniendo todo.

Descarga e instalación

Godot 4 no requiere instalación. Es un ejecutable portátil. Descárgalo desde godotengine.org. Elige la versión estándar (no .NET salvo que necesites C#).

  • Descargar Godot 4.3 o superior desde godotengine.org
  • Crear carpeta de proyecto (sin espacios ni tildes en la ruta)
  • Abrir Godot → New Project → seleccionar carpeta
  • Elegir "Mobile" o "Forward+" renderer (Forward+ para shaders avanzados)
  • Crear primera escena: Node2D como raíz
  • Guardar escena como main.tscn

    • Configuración para Pixel Art

    Antes de hacer cualquier sprite, configura el proyecto para pixel art:

    project.godot Project Settings → clave y valor 
    # Project Settings → Rendering → Textures
default_texture_filter = "Nearest"

# Project Settings → Display → Window
viewport_width  = 320   # resolución interna baja
viewport_height = 180
stretch_mode    = "canvas_items"   # escala pixel perfect
stretch_aspect  = "keep"
    ⚠️ Importante: Si no configuras Nearest como filtro de textura, los sprites tendrán antialiasing y se verán borrosos al escalar.

    GDScript // Variables y Tipos
    🔰 En palabras simples

    Una variable es una caja con nombre donde guardás un dato para usarlo después: la vida del jugador, su velocidad, si está vivo. El tipo indica qué clase de dato entra en esa caja: un número entero (int), uno con decimales (float), texto (String) o verdadero/falso (bool). Decir el tipo es opcional, pero ayuda a Godot a avisarte si te equivocás.

    Analogía: una variable es un casillero etiquetado: 'vida' guarda 100, 'nombre' guarda "Link". El tipo es el cartel que dice qué se puede meter adentro.

    GDScript variables.gd 
    extends Node

# ── Variables tipadas (recomendado en Godot 4) ──
var nombre: String  = "Link"
var vida: int      = 100
var velocidad: float = 150.0
var activo: bool  = true

# ── Constantes ──
const GRAVEDAD: float = 980.0
const MAX_VIDA: int   = 100

# ── Tipos de Godot ──
var posicion: Vector2  = Vector2(100, 200)
var color: Color      = Color.RED
var nodo_ref: Node2D   # referencia a nodo, null por defecto
var rect: Rect2        = Rect2(0, 0, 64, 64)

# ── @export: expone la variable en el Inspector ──
@export var jump_force: float = 400.0
@export var spritesheet: Texture2D

# ── @onready: asigna referencia cuando el nodo entra al árbol ──
@onready var sprite: AnimatedSprite2D = $AnimatedSprite2D
@onready var audio: AudioStreamPlayer2D = $SFX

    GDScript // Funciones
    🔰 En palabras simples

    Una función es un bloque de instrucciones con nombre que ejecutás cuando querés. En vez de repetir las mismas líneas una y otra vez, las metés en una función y la “llamás” por su nombre. Godot, además, llama solo a ciertas funciones especiales en momentos clave: _ready() al empezar y _process() en cada cuadro del juego.

    Analogía: una función es como el botón “picar” de la licuadora: lo apretás y se hace todo el proceso por vos, sin tener que explicar cada paso cada vez.

    GDScript funciones.gd 
    extends Node

# ── Funciones del ciclo de vida (built-in) ──
func _ready() -> void:
    # Se llama una vez cuando el nodo entra al SceneTree
    print("¡Nodo listo!")

func _process(delta: float) -> void:
    # Se llama cada frame. delta = tiempo desde el frame anterior
    # Ideal para lógica visual, no para física
    pass

func _physics_process(delta: float) -> void:
    # Se llama a ritmo fijo (60fps). Ideal para física y movimiento
    pass

func _input(event: InputEvent) -> void:
    # Captura cualquier input del jugador
    if event.is_action_pressed("jump"):
        saltar()

# ── Función personalizada con retorno ──
func calcular_dano(base: int, multiplicador: float = 1.0) -> int:
    return int(base * multiplicador)

# ── Lambda (Callable) ──
var on_die: Callable = func(): print("muerto")

# ── Funciones estáticas ──
static func distancia(a: Vector2, b: Vector2) -> float:
    return a.distance_to(b)

func saltar() -> void:
    print("¡Salto!")
    ℹ️ Usa _physics_process para movimiento del jugador. Usa _process para cosas visuales como tweens o UI. Mezclarlos puede causar jitter.

    GDScript // Control de Flujo
    🔰 En palabras simples

    El control de flujo hace que tu juego decida y repita. “Si la vida llega a 0, morí” es una decisión (if). “Repetí esto para cada enemigo” es un bucle (for). Sin esto, el código correría de arriba a abajo una sola vez; con esto, reacciona a lo que pasa y puede repetir tareas.

    Analogía: es como las indicaciones de un GPS: “si hay tráfico, doblá”, “seguí derecho hasta el tercer semáforo”. El programa elige el camino según la situación.

    GDScript control_flujo.gd 
    # ── if / elif / else ──
if vida <= 0:
    morir()
elif vida < 20:
    estado_critico()
else:
    estado_normal()

# ── match (equivale al switch) ──
match estado:
    "idle":
        update_idle()
    "run":
        update_run()
    "jump", "fall":    # múltiples valores
        update_air()
    _:                 # default
        update_idle()

# ── for ──
for i in range(5):        # 0,1,2,3,4
    print(i)

for enemigo in get_tree().get_nodes_in_group("enemies"):
    enemigo.take_damage(10)

# ── while ──
while municion > 0:
    disparar()
    municion -= 1

# ── Operador ternario ──
var texto = "vivo" if vida > 0 else "muerto"

    GDScript // Colecciones
    🔰 En palabras simples

    A veces no querés guardar un solo dato sino una colección. Un Array es una lista ordenada (elemento 0, 1, 2…), ideal para un inventario. Un Diccionario guarda pares clave: valor, como una agenda: buscás por un nombre y obtenés su dato ("vida" → 100).

    Analogía: el Array es una fila de casilleros numerados; el Diccionario es una agenda donde buscás por nombre, no por número.

    GDScript colecciones.gd 
    # ── Array ──
var items: Array[String] = ["espada", "escudo", "pocion"]
items.append("llave")       # agregar al final
items.erase("escudo")      # eliminar por valor
items.remove_at(0)        # eliminar por índice
print(items.size())        # tamaño
print(items.has("llave"))  # contiene

# ── Dictionary ──
var stats: Dictionary = {
    "vida": 100,
    "ataque": 15,
    "defensa": 5
}
stats["velocidad"] = 200       # agregar/modificar
print(stats.get("vida", 0))    # con valor por defecto
stats.erase("defensa")          # eliminar clave

# ── Iterar diccionario ──
for clave in stats:
    print(clave, ": ", stats[clave])

# ── Útil en metroidvania: inventario ──
var habilidades: Dictionary = {

    Nodos y Escenas // Señales (Signals)
    🔰 En palabras simples

    Una señal es un aviso que un nodo lanza cuando pasa algo (“me golpearon”, “entró un cuerpo”). Otros nodos pueden escuchar ese aviso y reaccionar, sin que el que avisa sepa quién lo escucha. Esto mantiene tu código ordenado: las piezas no dependen unas de otras directamente.

    var dir = -1 if anim.flip_h else 1
    velocity.x = dir * 600
    # Resetear después del cooldown
    await get_tree().create_timer(dash_cooldown).timeout

# ── Coyote time y jump buffer (avanzado) ──
var coyote_time: float  = 0.12  # segundos tras caer del borde
var jump_buffer: float  = 0.1   # buffer de salto anticipado
var coyote_timer: float = 0.0
var buffer_timer: float = 0.0

func _handle_jump(delta: float) -> void:
    if is_on_floor(): coyote_timer = coyote_time
    else: coyote_timer -= delta

    if Input.is_action_just_pressed("jump"): buffer_timer = jump_buffer
    else: buffer_timer -= delta

    if buffer_timer > 0 and coyote_timer > 0:
        velocity.y = jump_velocity
        buffer_timer = 0
        coyote_timer = 0

    Pixel Art // Shaders básicos
    🔰 En palabras simples

    Un shader es un mini-programa que corre en la placa de video y cambia cómo se ve algo: pintar un contorno, hacer parpadear en blanco al recibir daño. Suena avanzado, pero acá usás recetas cortas y listas para copiar y pegar.

    Analogía: como un filtro de Instagram aplicado a un sprite: la imagen es la misma, pero el shader cambia su apariencia en tiempo real.

    GLSL / GDShaderoutline.gdshader (contorno sprite)
    shader_type canvas_item;

uniform vec4 outline_color: source_color = vec4(1.0, 1.0, 1.0, 1.0);
uniform float outline_width: hint_range(0.0, 5.0, 1.0) = 1.0;

void fragment() {
    vec4 col = texture(TEXTURE, UV);
    float alpha = col.a;

    // Samplear píxeles adyacentes
    float ox = outline_width * TEXTURE_PIXEL_SIZE.x;
    float oy = outline_width * TEXTURE_PIXEL_SIZE.y;
    float a = texture(TEXTURE, UV + vec2( ox,  0)).a
            + texture(TEXTURE, UV + vec2(-ox,  0)).a
            + texture(TEXTURE, UV + vec2( 0,  oy)).a
            + texture(TEXTURE, UV + vec2( 0, -oy)).a;

    // Si el pixel es transparente pero tiene vecino opaco = contorno
    if (alpha < 0.5 && a > 0.0) {
        COLOR = outline_color;
    } else {
        COLOR = col;
    }
}
    GLSL / GDShaderhit_flash.gdshader (flash blanco al recibir daño)
    shader_type canvas_item;

uniform float flash_amount: hint_range(0.0, 1.0) = 0.0;

void fragment() {
    vec4 col = texture(TEXTURE, UV);
    // Mezcla hacia blanco según flash_amount
    col.rgb = mix(col.rgb, vec3(1.0), flash_amount * col.a);
    COLOR = col;
}

    Sistemas // Save / Load
    🔰 En palabras simples

    Guardar partida es escribir en un archivo los datos importantes (posición, vida, habilidades) para recuperarlos después. Cargar es leer ese archivo y dejar el juego como estaba. En Godot esto va en user://, una carpeta segura reservada para los datos del jugador.

const SAVE_PATH = "user://savegame.json"

func save() -> void:
    var data = {
        "player_pos":  [player.global_position.x, player.global_position.y],
        "health":      player.health,
        "abilities":   AbilitySystem.abilities,
        "current_room":RoomManager.current_room_path,
        "timestamp":   Time.get_datetime_string_from_system()
    }
    var file = FileAccess.open(SAVE_PATH, FileAccess.WRITE)
    file.store_string(JSON.stringify(data, "\t"))
    file.close()
    print("💾 Partida guardada")

func load_game() -> bool:
    if not FileAccess.file_exists(SAVE_PATH): return false
    var file  = FileAccess.open(SAVE_PATH, FileAccess.READ)
    var text  = file.get_as_text()
    file.close()
    var data = JSON.parse_string(text)
    if typeof(data) != TYPE_DICTIONARY: return false

    # Restaurar estado
    var p = data["player_pos"]
    player.global_position = Vector2(p[0], p[1])
    player.health          = data["health"]
    AbilitySystem.abilities = data["abilities"]
    return true

@onready var music_player: AudioStreamPlayer = $MusicPlayer

func play_music(stream: AudioStream, fade: float = 1.0) -> void:
    if music_player.playing:
        var t = create_tween()
        t.tween_property(music_player, "volume_db", -40, fade)
        await t.finished
    music_player.stream = stream
    music_player.volume_db = -40
    music_player.play()
    var t2 = create_tween()
    t2.tween_property(music_player, "volume_db", 0, fade)

    p.pitch_scale = pitch + randf_range(-0.05, 0.05)  # variación leve
    p.bus = "SFX"
    p.play()
    p.finished.connect(p.queue_free)

    extends Node

enum GameState { PLAYING, PAUSED, MENU, CUTSCENE, GAME_OVER }

var state: GameState = GameState.PLAYING

signal state_changed(new_state: GameState)

func set_state(new_state: GameState) -> void:
    if state == new_state: return
    state = new_state
    state_changed.emit(new_state)
    get_tree().paused = new_state == GameState.PAUSED

func pause() -> void:
    set_state(GameState.PAUSED)

func resume() -> void:
    set_state(GameState.PLAYING)

        State.JUMP:   _state_jump(d)
        State.ATTACK: _state_attack(d)
